XRD Digitizer v1.1 — 평가 방법론 정리 (연구용)
==============================================

이 문서는 코드 기준이다. 핵심 구현: eval/metrics.py, eval/gates.py, eval/report.py,
eval/gt_compat.py, scripts/run_eval_plan.py, eval/report.py CLI.


1. 평가가 무엇을 비교하는가
---------------------------
- **엔진 산출물**: 배치 실행 후 샘플별
    - `<output_dir>/<sample_id>_result.json`  (two_theta, intensities, …)
    - `<output_dir>/debug_<sample_id>/debug.json` (plot_box, calibration, postprocess, trace, candidate_stats, …)
- **정답(GT)**: 매니페스트 `gt_path` 가 가리키는 `<sample_id>_gt.json`
- GT에 `peaks` 배열이 없고 `peak_x_values` 만 있으면, eval 직전 **normalize_gt_for_eval** 이
  `peaks = [{"two_theta": x}, …]` 형태로 보강한다 (eval/gt_compat.py).


2. 배치·게이트 종류
-------------------
- **매니페스트 CSV**: `sample_id`, `gt_path`, 이미지 경로 열(clean_image_path 등).
- **phase-b** (scripts/run_eval_plan.py): clean / styled / real_like 도메인별로
  같은 파이프라인 설정으로 배치를 돌린 뒤, **게이트 타입별로** eval 리포트를 각각 만든다.
    - clean   → `--gate-type clean`    → report_clean.json
    - styled  → `--gate-type styled`   → report_styled.json
    - real_like → `--gate-type real_like` → report_real_like.json
- **한 샘플의 gate 판정**: `main` 지표 6개가 **모두** 임계값을 만족하면 `passed=True` (AND).


3. Main 지표 (게이트에 사용) — eval/metrics.py::_compute_main_metrics
----------------------------------------------------------------------

3.1 curve_y_mae_px
    - 의미: GT `pixel_curve_path` 와 **엔진이 복원한 곡선**을 픽셀 y로 비교한 평균 절대 오차(픽셀).
    - 정렬: `result["two_theta_values"]` 와 `intensities` 각 점에 대해, calibration의
      `x_scale`, `x_offset` 으로 픽셀 열 `col = round((two_theta - x_offset)/x_scale)` 을 구하고,
      GT 맵 `gt_map[col] = pt[1]` (원본 이미지 y) 과 매칭되는 열만 사용.
    - traced 픽셀 y: `(intensity - y_offset) / y_scale` 후 `plot_box[1]` 을 빼 ROI 기준으로 맞춤.
    - GT y도 `gt_py = gt_map[col] - plot_box[1]`.
    - **MAE**: 매칭된 열들에서 `|traced_py - gt_py|` 평균. 매칭 없으면 999.
    - 주의: x가 격자에 정확히 안 맞으면 열 매칭이 흔들릴 수 있음.

3.2 major_peak_x_error
    - 의미: 검출된 **주요 피크**들의 2θ 위치가 GT 피크 x와 얼마나 떨어졌는지의 평균을 **픽셀 단위**로 표현.
    - GT: `gt["peaks"]` 각 원소의 `two_theta` (또는 리스트면 [0]).
    - 검출: `debug["calibration"]["peak_positions_2theta"]` 중 `is_major=True` 인 항목만.
    - 없으면 (999, 999) 반환.
    - 각 major에 대해 `min_g |two_theta_det - gx|` (2θ 차이)를 구한 뒤, `*(plot_box 폭)/(x_max-x_min)` 으로 **도 단위 차이 → 픽셀** 환산한 값들의 **평균**.
    - **major_peak_y_error**: 현재 구현상 항상 0.0 (자리만 있음).

3.3 peak_recall
    - 의미: GT 피크 개수 대비, **검출 피크가 2θ에서 얼마나 맞았는지**의 비율 (recall 쪽 정의).
    - tolerance: `x_range * 0.02` 즉 **x축 전 범위의 2%** 이내면 “맞춘 것”.
    - 검출 위치: `peak_positions_2theta` 의 모든 피크 (major만이 아님).
    - 각 GT 피크 gx에 대해 `any(|dx - gx| <= tolerance)` 이면 matched.
    - 반환: `matched / len(gt_peaks)`.

3.4 max_gap_px
    - 의미: gap_fill 로 메운 구간 중 **가장 긴 연속 결측 길이**(열 개수).
    - `debug["postprocess"]["gap_fill"]["gap_ranges"]` 각 `(gs, ge)` 에 대해 `ge - gs` 의 최대값.
    - 없으면 0.

3.5 calibration_roundtrip_error
    - 의미: `debug["calibration"]["roundtrip_error"]["total_mean_error_px"]` (없으면 0).
    - 축 매핑 왕복 오차의 요약치로 게이트에 사용.


4. Debug / Diagnosis 지표 (게이트에는 안 씀, 실패 라벨에 씀)
------------------------------------------------------------
- **debug** (eval/metrics.py::_compute_debug_metrics): IoU, valid_ratio, trace_score,
  tail_mae_px, tail_collapse_rate, peak_precision, peak_f1, prominence_preservation 등.
- **diagnosis** (_compute_diagnosis_metrics): candidate_recall_per_column, empty_column_rate,
  recovery_success_rate, reentry_count, path_margin_instability.


5. 수락 게이트 임계값 — eval/gates.py::GATES
--------------------------------------------
지표명              clean      styled    real_like   비고
------------------------------------------------------------------
curve_y_mae_px  ≤ 5.0      ≤ 6.0     ≤ 8.0       낮을수록 좋음
major_peak_x_error ≤ 4.0 ≤ 6.0     ≤ 8.0       픽셀
major_peak_y_error ≤ 6.0 ≤ 8.0     ≤ 10.0      현재 항상 0
peak_recall     ≥ 0.70     ≥ 0.68    ≥ 0.60      높을수록 좋음 (RECALL_IS_GTE)
max_gap_px      ≤ 10       ≤ 14      ≤ 18
calibration_roundtrip_error ≤ 1.0 ≤ 1.8     ≤ 2.5

- **peak_recall** 만 “이상 통과”, 나머지는 “이하 통과”.
- 샘플 gate: 위 6개 **전부** 만족 → pass.


6. 배치 리포트 집계 — eval/report.py::_aggregate
-------------------------------------------------
- **pass_rate** = (샘플 중 gate.passed 인 개수) / 샘플 수.
- **main_metrics_stats**: 샘플별 main 값들에 대해 mean / std / min / max.
- **failure_taxonomy_counts**: 샘플별 `label_failures` 문자열 라벨을 전 샘플에 걸쳐 카운트.


7. 실패 자동 라벨 — eval/gates.py::label_failures
--------------------------------------------------
(게이트 통과 여부와 별개로, 원인 힌트 태그를 붙인다. 여러 개 동시에 붙을 수 있음.)

- candidate_starvation: empty_column_rate > 0.05
- wrong_branch_lock_in: path_margin_instability > 0.6
- tail_collapse: tail_mae_px > 8 또는 tail_collapse_rate > 0.3
- calibration_mismatch: main calibration_roundtrip_error > 1.0
- peak_miss_after_smoothing: peak_f1 < 0.5 이고 peak_recall ≥ 0.6
- text_intrusion: IoU < 0.7 이고 empty_column_rate < 0.03
- grid_confusion: candidate_recall_per_column > 0.98 이고 curve_y_mae_px > 10
- legend_capture: recovery_success_rate < 0.5 이고 reentry_count > 0

→ 임계값은 휴리스틱이며, 실제 근본 원인과 1:1 대응은 아닐 수 있다.


8. 실행 방법 (요약)
--------------------
- 단일 게이트 배치 평가:
    python3 eval/report.py --manifest_csv … --output_dir … --gate_type clean --report_json …
- phase-b 전체 (배치 + 3종 리포트):
    python3 scripts/run_eval_plan.py --repo-root . phase-b --max-samples N --tag mytag
- 플래그 조합은 batch_run / phase-b 인자로 전달 (--axis-mask-margin, --use-ridge-candidates, --peak-single-pass).


9. 해석 시 알면 좋은 점
-----------------------
- **게이트는 “제품 출하” 수준에 가깝게 잡혀 있을 수 있다.** 현재 파이프라인 평균 MAE(~13px)와
  clean 곡선 임계(5px) 사이 간격이 크면 pass_rate 0%에 가깝게 나오는 것이 자연스럽다.
- **peak_recall** 과 **debug peak_f1 기반 라벨**은 정의가 다르다 (전자는 tolerance 매칭, 후자는 PRF).
- **major_peak_x_error** 는 major만 보고, **peak_recall** 은 전체 검출 피크로 매칭한다.
- 비교 연구 시: 동일 `max_samples`, 동일 매니페스트, 동일 `--tag` 출력만 바꿔 **상대 비교**를 권장.
